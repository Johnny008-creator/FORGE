# v0.7.0 — The Modular Framework Update (2026-04-21)

This version represents the most significant architectural shift since the project's inception. Forge has transformed from a monolithic script into a full-fledged, extensible framework.

## ✨ Key Highlights

### 🏗️ Modular Architecture
The codebase has been partitioned into logical layers to facilitate easier maintenance and eliminate development errors:
- **Core**: Agent brain, context management, and JSON repair logic.
- **Providers**: Abstraction layer for AI backends (Ollama, OpenAI ready).
- **Tools**: Independent modules for each tool with dynamic registration.
- **UI**: Decoupled themes, inputs, and outputs powered by Rich.
- **Config**: Centralized settings management and environment validation.

### 📡 AI Provider System
Introduced the `BaseProvider` abstract class. This allows Forge to communicate with any API (e.g., Anthropic, OpenAI) in the future while maintaining the same user interface.

### 🛡️ Resilience & Hardening (Tiny Model Focus)
- **Argument Filtering**: Tools now automatically filter hallucinated arguments from smaller models using function signature inspection.
- **Path Guard**: Protection against root-directory write attempts without a specified path.
- **Infinite Loop Hard-Stop**: Enhanced detection and termination of repeating model behaviors.

### 🪝 Hook API & Plugins
Ready for an ecosystem of extensions. Forge now supports events like `before_tool` or `on_response`, which external plugins can subscribe to.

## 📁 New Repository Structure
```text
core/        - Agent brain & Loop
providers/   - AI Backend abstraction
tools/       - Modular tool functions
ui/          - Rich UI components
tiers/       - Model-specific profiles
config/      - System settings
plugins/     - Extension system
tests/       - Pytest suite
```

## 🧪 Validation
Added a comprehensive test suite in `tests/` covering parsing logic, tool execution, and tier management.
