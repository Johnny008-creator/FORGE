# Forge v0.6.0 — The "Claude Code" Update

## 🌟 Overview
Version 0.6.0 marks the most significant visual and structural upgrade to Forge since its inception. Inspired by professional CLI agents like Claude Code, this release focuses on transparency, interactivity, and extreme reliability for local LLMs.

---

## ✨ Key Features

### 🖥️ Professional UI/UX
- **The "Flower" Animation**: A custom Unicode-based spinner that remains active even during LLM processing delays.
- **Dynamic Headers**: Real-time display of the current model and elapsed thinking time.
- **Live Footer**: Heuristic-based token velocity tracking (t/s) and total response count.
- **Dashboard Footer**: A permanent status bar showing your working directory, execution mode, and current model.

### 🧠 Intelligent Decision Making (`ask_choice`)
Forge can now stop and ask for your input when faced with multiple valid paths. This prevents "hallucinated" file creations and gives you full control over the agent's architectural decisions.

### ⚡ Auto-Ollama Backend
No more "Connection Refused" errors. Forge now detects if Ollama is running and can automatically launch the server in a hidden background process (Windows/Linux support included).

### 📊 Persistent Economics
Forge now calculates how much you've saved by running locally! 
- Data stored in `~/Documents/FORGE_agent_memory/usage_stats.json`.
- Visual dashboard accessible via `/usage`.

---

## 🔧 Technical Improvements
- **Prompt Engineering**: Transitioned to a strict "Analyze -> Plan -> Act" workflow in the system prompt.
- **Self-Correction**: Improved error parsing allows the agent to read tool failures and automatically try a different approach.
- **Sub-3B Optimization**: Special "Tiny Mode" prompt profiles to prevent smaller models (like Qwen 0.5B/1.5B) from looping in decision menus.

---

## 🚀 Installation & Upgrade
If you have Forge already cloned, simply pull the latest and run:
```bash
pip install -e .
```
You can then run Forge from anywhere using the `forge` command.
