# Forge v0.3.0 Upgrade Summary

## What Changed

### 1. Execution Modes (Auto/Ask/Manual)
- **`/auto`** — Execute all tool calls without confirmation (fastest, less safe)
- **`/ask`** — Default: safe tools auto-execute, destructive ops ask (balanced)
- **`/manual`** — Confirm every single tool execution (safest, slowest)
- **`/mode`** — Show current mode

Status bar now displays the active mode.

### 2. Robust Tool Call Parsing
Model outputs are parsed in **3 strategies**:

1. **Standard JSON** (best for smart models):
   ```json
   {"tool":"read","args":{"path":"forge.py"}}
   ```

2. **Function calls** (good for tiny models):
   ```python
   read("forge.py")
   shell("ls -la", 30)
   list()
   ```

3. **Markdown code blocks** (friendly for all models):
   ```markdown
   ```json
   {"tool":"read","args":{"path":"test.py"}}
   ```
   ```

**Why this matters**: Tiny models (0.5B) don't always output perfect JSON. Now we accept multiple formats.

### 3. Few-Shot Examples in System Prompt
Tiny models get concrete examples:
```
EXAMPLES OF TOOL CALLS:

User: list files
You: {"tool":"list","args":{}}

User: read setup.py
You: {"tool":"read","args":{"path":"setup.py"}}

User: run tests
You: {"tool":"shell","args":{"cmd":"python -m pytest"}}
```

This **massively improves** tool-calling reliability on 0.5B-3B models.

## Code Changes

### New Global State
```python
EXEC_MODE = "ask"           # Current execution mode
SAFE_TOOLS = {              # Never require confirmation
    "read", "list", "search"
}
```

### New Functions
```python
should_confirm(tool_name: str) -> bool
    # Checks if tool needs confirmation based on EXEC_MODE

ask_confirm(tool_name: str, args: dict) -> bool
    # Unified confirmation prompt for any tool
```

### Enhanced Functions
```python
extract_tool_calls(text: str) -> list
    # Now uses 3-strategy parsing instead of single regex

build_system_prompt(workdir, tiny=False) -> str
    # Now includes few-shot examples

agentic_loop(...)
    # Uses should_confirm() instead of shell-specific logic
```

### New Slash Commands
```
/mode       Show current execution mode
/auto       Switch to auto mode
/ask        Switch to ask mode (default)
/manual     Switch to manual mode
```

## Known Limitations

- Context window small for tiny models (4 msgs for 0.5B)
- Token limits force `/clear` periodically on tiny models
- No persistent session memory yet
- No parallel tool execution
- No custom tool definitions
